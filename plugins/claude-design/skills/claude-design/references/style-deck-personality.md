# 슬라이드 덱 스타일 성격 가이드

> **사용 목적**: 스타일 추천 시 단순 mood/industry 태그 매칭만으로는 부족하다.
> 이 파일은 각 스타일이 슬라이드 덱으로 생성됐을 때 가지는 **발표 맥락 적합성**을 정의한다.
>
> STEP A에서 스타일 후보 top 3 결정 후, 각 후보의 성격이 콘텐츠 목적과 맞는지 이 파일로 검증한다.

---

## 성격 프레임워크

각 스타일은 아래 3개 축으로 슬라이드 덱 맥락을 정의한다:

| 축 | 설명 |
|----|------|
| **발표 성격** | 이 스타일로 만든 덱이 청중에게 전달하는 인상 (권위 / 설득 / 감성 / 분석 / 혁신 등) |
| **최적 발표 유형** | 어떤 발표 맥락에 가장 자연스러운가 |
| **공간 문법** | 슬라이드의 구조를 어떻게 만드는가 — 타이포 주도 / 면 분할 / 플로팅 / 선 분할 |

---

## Light 스타일 성격

### Raw Form
- **발표 성격**: 브랜드 정체성이 뚜렷한, 날 것의 정직함, 독립적
- **최적 발표 유형**: 브랜드 덱, 포트폴리오, 디자인 에이전시 제안, 아이덴티티 발표
- **공간 문법**: 여백과 텍스트가 구조를 만든다. 패널 면분할과 선분할 혼용.
- **피해야 할 맥락**: 보수적인 기업 투자자 앞 피치, 공공기관 보고서
- **콘텐츠 신호**: "브랜드 스토리", "디자인 철학", "우리 팀의 방식", "포트폴리오"

### Swiss Style (Swiss Minimalist)
- **발표 성격**: 권위적, 정보 주도적, 타이포그래피가 전달하는 신뢰
- **최적 발표 유형**: 컨퍼런스 발표, 학술/연구 결과 공유, 기업 내부 정책 보고
- **공간 문법**: 타이포 주도. 굵은 서체가 레이아웃 분할선이 된다. 선 분할 사용.
- **피해야 할 맥락**: 소비자 대상 마케팅 발표, 감성적 브랜드 스토리
- **콘텐츠 신호**: "데이터 기반", "분석 결과", "정책 제안", "인사이트 공유", "연구"

### Corporate Trust
- **발표 성격**: 안정감, 예측 가능한 신뢰, 전문성
- **최적 발표 유형**: VC/투자자 피치, 기업 파트너십 제안, 영업 프레젠테이션, 연간 보고서
- **공간 문법**: 패널 면분할. 명확한 구획이 신뢰를 만든다. 40:60 비율 기본.
- **피해야 할 맥락**: 창의적 아이디어 발표, 브랜드 실험적 캠페인
- **콘텐츠 신호**: "투자 유치", "파트너십", "기업 실적", "제안서", "계약"

### Bold Editorial Design
- **발표 성격**: 대담한 관점 제시, 아이디어가 주인공
- **최적 발표 유형**: 마케팅 전략 발표, 트렌드 리포트, 캠페인 제안, 오피니언 공유
- **공간 문법**: 타이포 + 면 분할 혼용. 텍스트 크기 대비가 극명하다.
- **피해야 할 맥락**: 기술 사양 발표, 보수적 기업 보고서
- **콘텐츠 신호**: "트렌드", "우리가 믿는 것", "캠페인", "크리에이티브 방향", "왜 우리인가"

### Warm Industrial Gray
- **발표 성격**: 직관적, 실용적, 프레임보다 내용이 강조됨
- **최적 발표 유형**: 프로덕트 데모, UX/UI 리뷰, 내부 프로젝트 업데이트, 팀 OKR
- **공간 문법**: 여백 분할 + 선 분할. 내용이 공간을 채운다.
- **피해야 할 맥락**: 외부 투자자 피치, 감성적 브랜드 스토리
- **콘텐츠 신호**: "제품 현황", "기능 업데이트", "팀 공유", "스프린트 리뷰", "로드맵"

### Tech Editorial
- **발표 성격**: 분석적, 정교한, 데이터와 인사이트가 주인공
- **최적 발표 유형**: 테크 트렌드 분석, 마켓 리서치 발표, 기술 전략 보고
- **공간 문법**: 타이포 주도 + 선 분할. 제목이 섹션을 구조화한다.
- **피해야 할 맥락**: 감성적 브랜드 캠페인, 투자자 감성 호소 피치
- **콘텐츠 신호**: "시장 분석", "기술 트렌드", "경쟁 환경", "데이터로 보는"

### Neo-Brutalism (Light)
- **발표 성격**: 직접적, 비타협적, 메시지를 숨기지 않는다
- **최적 발표 유형**: 혁신 제안, 스타트업 내부 공유, 문제 정의 발표
- **공간 문법**: 강한 면 분할 + 명확한 경계. 색상 대비가 극적이다.
- **피해야 할 맥락**: 보수적 기업 제안서, 럭셔리 브랜드 발표
- **콘텐츠 신호**: "문제는 이것이다", "직접 말하면", "기존 방식의 실패", "우리의 대안"

### Architectural Type System
- **발표 성격**: 체계적, 시스템 사고, 기술적 정밀함
- **최적 발표 유형**: 개발자/엔지니어링 발표, 시스템 아키텍처 설명, 기술 스펙 공유
- **공간 문법**: 타이포 주도 + 그리드 기반 선 분할. 여백이 정보 계층을 만든다.
- **피해야 할 맥락**: 소비자 감성 호소, 마케팅 캠페인
- **콘텐츠 신호**: "아키텍처", "기술 스택", "시스템 설계", "API", "인프라"

### Minimalist Modern
- **발표 성격**: 명료함, 군더더기 없음, 핵심만 남긴다
- **최적 발표 유형**: 스타트업 피치, 제품 컨셉 발표, 간결한 전략 공유
- **공간 문법**: 여백 분할. 공간이 메시지다.
- **피해야 할 맥락**: 복잡한 데이터 발표, 상세 기술 설명
- **콘텐츠 신호**: "핵심은", "단 하나의 메시지", "집중", "why → what → how"

### Poster Modernist
- **발표 성격**: 시각적 강렬함, 하나의 아이디어가 슬라이드를 지배
- **최적 발표 유형**: 브랜드 론칭, 컨셉 소개, 키노트형 발표
- **공간 문법**: 전면 타이포 + 여백. 슬라이드당 하나의 큰 메시지.
- **피해야 할 맥락**: 세부 정보 전달, 데이터 중심 보고
- **콘텐츠 신호**: "론칭", "발표", "비전", "하나의 아이디어로"

## Dark 스타일 성격

### Midnight Editorial
- **발표 성격**: 프리미엄, 감성적 깊이, 공간이 이야기를 만든다
- **최적 발표 유형**: 브랜드 스토리, 크리에이티브 에이전시 발표, 비전 선언
- **공간 문법**: 플로팅 콘텐츠. 단일 다크 그라운드 위에 요소가 떠 있다.
- **피해야 할 맥락**: 수치 중심 투자 보고, 기술 스펙 발표
- **콘텐츠 신호**: "우리가 만드는 세계", "브랜드 여정", "감성적 스토리", "비전"

### Neural Noir
- **발표 성격**: AI/기술의 혁신성 + 럭셔리 신뢰
- **최적 발표 유형**: AI 스타트업 피치, 기술 혁신 발표, 프리미엄 B2B 제안
- **공간 문법**: 플로팅 + 면 분할 혼용. 어두운 배경 위 광도 높은 요소.
- **피해야 할 맥락**: 교육/연구 발표, 소비자 친화적 제품 소개
- **콘텐츠 신호**: "AI 기술", "혁신 솔루션", "프리미엄 B2B", "미래 기술 발표"

### Minimalist Beta
- **발표 성격**: 결과만 말한다, 군더더기 제로
- **최적 발표 유형**: 데모 데이, 내부 전략 보고, 스타트업 초기 피치
- **공간 문법**: 여백 분할. 다크 배경 위 최소한의 텍스트.
- **피해야 할 맥락**: 감성적 브랜드 발표, 시각 중심 마케팅
- **콘텐츠 신호**: "데모", "수치", "검증된 결과", "단계별 실행"

### Dark Matte Editorial
- **발표 성격**: 진지함, 깊은 전문성, 과장 없는 사실
- **최적 발표 유형**: 컨설팅/전략 발표, 기업 심층 분석, Annual Report
- **공간 문법**: 여백 분할 + 선 분할. 어두운 배경이 집중을 유도한다.
- **피해야 할 맥락**: 스타트업 에너지 피치, 소비자 광고 형태 발표
- **콘텐츠 신호**: "전략 보고", "컨설팅 제안", "심층 분석", "연간 성과"

### Cyber Serif
- **발표 성격**: 기술 전문가를 위한, 정확성과 미래 방향의 조화
- **최적 발표 유형**: 개발자 컨퍼런스, 기술 로드맵 발표, 플랫폼 아키텍처
- **공간 문법**: 타이포 주도. 세리프 서체가 기술적 신뢰를 준다.
- **피해야 할 맥락**: 일반 소비자 발표, 감성 마케팅
- **콘텐츠 신호**: "기술 로드맵", "플랫폼", "개발자 발표", "시스템 구조"

## 스타일 선택 의사결정 트리

```
발표 목적은?
│
├─ 투자/설득 (VC, 파트너, 영업)
│   ├─ 기술 스타트업 → Neural Noir / Minimalist Beta
│   ├─ 기업형 → Corporate Trust
│   └─ 크리에이티브 분야 → Bold Editorial Design / Raw Form
│
├─ 내부 공유 (팀, 경영진)
│   ├─ 제품/개발 → Warm Industrial Gray / Architectural Type System
│   ├─ 전략/분석 → Swiss Style / Tech Editorial / Dark Matte Editorial
│   └─ 목표/OKR → Minimalist Modern / Minimalist Beta
│
├─ 브랜드/마케팅 (론칭, 캠페인)
│   ├─ 프리미엄 감성 → Midnight Editorial / Poster Modernist
│   ├─ 강렬한 주장 → Neo-Brutalism / Bold Editorial Design
│   └─ 브랜드 아이덴티티 → Raw Form
│
└─ 지식/교육 (컨퍼런스, 리서치)
    ├─ 데이터 중심 → Swiss Style / Tech Editorial
    ├─ 기술 전문 → Cyber Serif / Architectural Type System
    └─ 아이디어 공유 → Poster Modernist / Bold Editorial Design
```

## STEP A 통합 사용법

```
1. style-recommender.md로 mood/industry 기반 top 3 후보 선정
2. 이 파일에서 각 후보의 "발표 성격"과 "최적 발표 유형" 확인
3. 콘텐츠의 발표 목적 + 청중과 매칭 검증
4. 최종 추천 시 "이 스타일을 추천하는 이유"에 성격 매칭 설명 포함

추천 출력 예시:
① Raw Form
  "브랜드 정체성을 전면에 내세우는 스타일. 이 덱은 디자인 철학과
   팀의 방식을 설명하는 내용이라 '날 것의 정직함'을 전달하는
   Raw Form의 성격과 자연스럽게 연결됩니다."
```
