# 슬라이드 레이아웃 베리에이션 레퍼런스

> **목적**: 그리드 구조 × 구성 요소 조합으로 나올 수 있는 실제 베리에이션 카탈로그.  
> 에이전트가 레이아웃 선택 전 이 파일을 참조해 다양성을 확보한다.

---

## 0. 분할 표현 방식 (Border Expression)

레이아웃이 **"어디에 무엇을"** 결정한다면, 표현 방식은 **"경계를 어떻게 보이게 하는가"** 를 결정한다.  
같은 `2col-30` 레이아웃이라도 표현 방식에 따라 전혀 다른 스타일이 나온다.  
모든 그리드 유형에 직교(orthogonal)로 적용된다.

### 세 가지 방식

#### 면 분할 (Panel Fill)
경계 양쪽에 서로 다른 배경색이 있어 영역이 명확하게 구분된다.  
`background: var(--primary)` 또는 `background: var(--surface)` 로 패널을 채운다.  
강한 대비, 명확한 구획. 정보 밀도가 높은 슬라이드에 적합.  
주의: 모든 슬라이드가 면 분할이면 덱이 단조로워진다 — 전체의 40% 이하 권장.

#### 선 분할 (Rule Line)
배경은 동일하고 `border: 1px solid var(--border)` 또는 `border-left/right: 2~4px solid var(--accent)` 하나로만 구획한다.  
공간은 연속적이고 경계는 암시적. 세련되고 모던한 인상.  
Swiss International, Editorial, Linear 스타일에 특히 적합.

#### 여백 분할 (Whitespace)
선도 면도 없다. 콘텐츠의 위치, 크기, 정렬 방향이 시각적 구획을 만든다.  
배경 전체가 하나의 캔버스가 된다. 구획이 "보이지 않아야" 성공.  
Minimal, Midnight Editorial, Typography-Led 스타일. 숙련된 레이아웃 판단 필요.

### 비율 변형

`2-col` 계열은 비율을 반드시 명시한다. 숫자는 좌:우 기준.

| 비율 이름 | 좌:우 | flex 값 | 용도 |
|---------|-------|---------|------|
| `2col-20` | 20:80 | `flex:0 0 256px` + `flex:1` | 이미지·목업 중심 |
| `2col-30` | 30:70 | `flex:0 0 384px` + `flex:1` | 헤딩 좌 + 본문 우 (가장 일반적) |
| `2col-40` | 40:60 | `flex:0 0 512px` + `flex:1` | 균형 잡힌 2열, 좌측 약간 강조 |
| `2col-50` | 50:50 | `flex:0 0 640px` + `flex:1` | 완전 균등 — 비교·대칭 |
| `2col-30r` | 70:30 | `flex:1` + `flex:0 0 384px` | 역순 — 타이포 좌 + 액센트 우 |
| `2col-20r` | 80:20 | `flex:1` + `flex:0 0 256px` | 역순 — 비주얼 좌 + 라벨 우 |

> **역순(r) 사용 시점**: 콘텐츠 무게중심이 좌측에 있을 때. 커버에서 변화를 줄 때. 연속 슬라이드에서 단조로움을 깰 때.

---

### 그리드 × 표현 방식 조합 매트릭스

| 그리드 | 면 분할 | 선 분할 | 여백 분할 |
|-------|--------|--------|---------|
| `1col-full` | 솔리드 배경 전체 | — (단일 영역) | 여백이 구조 (Swiss 커버) |
| `2col-30` | 우측 패널 채움 (일반적) | 세로 룰 1개 | 타이포 크기 차이로 구분 |
| `2col-50` | 좌:우 각각 다른 색 | 중앙 세로 룰 | 좌:우 정렬 방향 역전 |
| `2col-30r` | 우측 좁은 액센트 패널 | 좌측 룰 | 우측 여백으로 처리 |
| `2row-30` | 상단 밴드 색 채움 | 가로 룰 1개 | 상단 작은 라벨만 |
| `band+col` | 밴드 배경색 다르게 | 밴드 하단 룰 | 패딩·크기 차이로 구분 |
| `diagonal` | 사선 clip-path 채움 | — (사선 선은 읽기 어려움) | — (사선 여백 구분 불가) |

> `diagonal`은 면 분할에서만 유효하다. 선·여백 방식으로는 사선 경계를 인식하기 어렵다.

---

## 1. `1col-full` — 1열 전체화면

공간 전체를 하나의 영역으로 사용. 가장 임팩트 강함.

| 변형 | 구성 요소 | 느낌 | 용도 |
|------|---------|------|------|
| A | 히어로 타이틀 (중앙) | 선언, 드라마틱 | 커버, 섹션 구분 |
| B | 히어로 타이틀 + 한 줄 서브 | 컨텍스트 있는 선언 | 커버, 챕터 |
| C | 전체 배경 사진 + 타이틀 오버레이 | 감성, 분위기 | 인트로, 전환 |
| D | 큰 숫자 하나 + 라벨 | 충격적 지표 | Statement |
| E | 인용구 풀스크린 | 고객 목소리 | Quote |
| F | 전체 다이어그램/차트 | 데이터 집중 | Data |

---

## 2. `1col-center` — 1열 센터 정렬

여백을 살리고 핵심 하나에 집중. 전체화면과 달리 주변 여백이 살아있음.

| 변형 | 구성 요소 | 느낌 | 용도 |
|------|---------|------|------|
| A | 타이틀 + 한 줄 설명 | 미니멀, 집중 | 핵심 메시지 |
| B | 큰 숫자 + 설명 라벨 | 명확한 지표 | Key Metric |
| C | 아이콘 + 타이틀 + 서브 | 친근한 집중 | 기능 하나 |
| D | 타이틀 + 프로세스 한 줄 (→ 연결) | 흐름 요약 | Philosophy |
| E | 타이틀 + 불렛 3개 (짧게) | 요약 집중 | Summary |

---

## 3. `2col-50` — 2열 균등 (50:50)

좌우 동등한 비중. 비교·대칭 구조에 최적.

| 변형 | 좌 칸 | 우 칸 | 느낌 | 용도 |
|------|------|------|------|------|
| A | 텍스트 단락 | 텍스트 단락 | 대칭, 균형 | 비교 설명 |
| B | 사진 | 텍스트 단락 | 비주얼+설명 | Before/After |
| C | 큰 숫자 | 큰 숫자 | 강렬한 대비 | 지표 비교 |
| D | 아이콘+설명 | 아이콘+설명 | 균형 나열 | 2가지 가치 |
| E | 다크 패널 타이틀 | 크림 패널 설명 | 구조적 대비 | Highlight |

---

## 4. `2col-30` — 2열 비균등 (30:70)

좌측 강조 헤딩 + 우측 본문. 가장 흔하지만 우측 컴포넌트로 차별화 가능.

| 변형 | 좌 칸 (30%) | 우 칸 (70%) | 느낌 | 용도 |
|------|-----------|-----------|------|------|
| A | 히어로 타이틀 | 단락 텍스트 | 설명형 | Highlight |
| B | 히어로 타이틀 | 불렛 리스트 | 요약형 | Feature |
| C | 히어로 타이틀 | Key Metrics 3종 | 지표 강조 | Stats |
| D | 히어로 타이틀 | 사진 | 비주얼 | Photo Split |
| E | 히어로 타이틀 | 타임라인 (세로) | 단계 표현 | Process |
| F | 히어로 타이틀 | 인용구 | 증거 제시 | Quote+Claim |
| G | 히어로 타이틀 | 2×2 소카드 | 나열 | Multi-feature |

---

## 5. `2col-20` — 2열 비균등 (20:80)

이미지 중심, 텍스트는 최소. 비주얼이 메인일 때.

| 변형 | 좌 칸 (20%) | 우 칸 (80%) | 느낌 | 용도 |
|------|-----------|-----------|------|------|
| A | 수직 텍스트 라벨 | 전체 사진 | 임팩트 사진 | Hero Photo |
| B | 컬러 액센트 바 | 목업 스크린 | 제품 쇼케이스 | App Showcase |
| C | 소제목+라벨 | 대형 차트 | 데이터 우선 | Chart Focus |

---

## 6. `2row-30` — 2행 비균등 (상30:하70)

상단 좁은 헤딩 밴드 + 하단 넓은 콘텐츠 영역. 풀와이드 활용 가능.

| 변형 | 상단 (30%) | 하단 (70%) | 느낌 | 용도 |
|------|----------|----------|------|------|
| A | 소제목+라벨 | 단락+불렛 | 계층 명확 | Highlight |
| B | 소제목+라벨 | 3열 아이콘+라벨 | 구조적 나열 | Feature 3종 |
| C | 소제목+라벨 | 가로 타임라인 | 흐름 표현 | Timeline |
| D | 소제목+라벨 | 대형 차트 | 데이터 집중 | Data |
| E | 소제목+라벨 | 이미지 풀와이드 | 사진 임팩트 | Photo |
| F | 소제목+라벨 | 목업 2~3개 나란히 | 제품 비교 | Mockup |

---

## 7. `2row-50` — 2행 균등 (상50:하50)

상하 동등한 영역. 흐름·단계 표현에 적합.

| 변형 | 상단 (50%) | 하단 (50%) | 느낌 | 용도 |
|------|----------|----------|------|------|
| A | 문제/배경 | 해결책 | 대비 구조 | Problem→Solution |
| B | 타이틀+서브 | 숫자 3종 강조 | 임팩트+근거 | Claim+Stats |
| C | 사진 | 캡션+설명 | 비주얼+텍스트 | Photo+Caption |

---

## 8. `3col-equal` — 3열 균등

3개 항목 병렬 나열. 좌우 대칭.

| 변형 | 각 칸 구성 요소 | 느낌 | 용도 |
|------|-------------|------|------|
| A | 아이콘+타이틀+설명 | 카드형 나열 | Feature 3종 |
| B | 큰 숫자+라벨+서브 | 지표 강조 | Key Metrics |
| C | 사진+캡션 | 비주얼 나열 | Gallery |
| D | 번호+타이틀+설명 | 단계 표현 | Process 3단계 |
| E | 인물사진+이름+역할 | 팀 소개 | Team |

---

## 9. `2x2` — 4분할 (2×2)

4개 항목 균등 배치. 상단에 전체 제목 행이 있는 버전도 가능.

| 변형 | 각 칸 구성 요소 | 느낌 | 용도 |
|------|-------------|------|------|
| A | 아이콘+타이틀+설명 | 카드형 4종 | Feature 4종 |
| B | 키워드+설명 한 줄 | 간결 나열 | 4가지 원칙 |
| C | X축/Y축 포지셔닝 | 포지셔닝 맵 | 2×2 Matrix |
| D | 사분면 텍스트+레이블 | 분석 프레임 | SWOT 등 |

---

## 10. `band+col` — 상단 밴드 + 하단 열 분할

상단: 제목/맥락 전달 / 하단: 세부 내용을 열로 분리.

| 변형 | 상단 밴드 | 하단 구조 | 느낌 | 용도 |
|------|---------|---------|------|------|
| A | 소제목+설명 | 2열 (설명+리스트) | 계층 강조 | Highlight |
| B | 소제목+설명 | 3열 아이콘+라벨 | 구조적 | Feature 3종 |
| C | 소제목+설명 | 4열 숫자+라벨 | 지표 나열 | Stats 4종 |
| D | 소제목+설명 | 가로 타임라인 | 흐름 | Timeline |
| E | 타이틀+서브 | 2열 (좌: 텍스트 / 우: 사진) | 복합 | Highlight+Photo |

---

## 11. `diagonal` — 대각선·사선 분할

기하학적 분할. 역동성, 강한 브랜드 인상.

| 변형 | 구성 | 느낌 | 용도 |
|------|------|------|------|
| A | 좌상 다크 삼각형 + 우하 크림 | 방향성 | Cover, Section |
| B | 우상→좌하 대각 컬러 밴드 | 에너지 | Statement |
| C | 비스듬한 컬러 패널 + 텍스트 | 역동적 | Section Title |

---

## 12. `quarter` — 1/4 패널 강조

한쪽 1/4만 강한 컬러·요소로 채우고 나머지는 여백 또는 텍스트.

| 변형 | 1/4 영역 | 3/4 영역 | 느낌 | 용도 |
|------|---------|---------|------|------|
| A | 좌측 컬러 액센트 바 | 우측 텍스트+여백 | 세련, 미니멀 | Quote, Statement |
| B | 하단 컬러 밴드 | 상단 히어로 타이틀 | 안정감 | Closing |
| C | 우측 이미지 패널 | 좌측 텍스트 | 비대칭 임팩트 | Feature |

## 조합 매트릭스 — 10장 덱 예시

동일 콘텐츠를 그리드 다양성 규칙 적용 시:

| # | 슬라이드 역할 | 그리드 | 주요 컴포넌트 |
|---|-------------|-------|------------|
| 1 | Cover | `2col-30` | 히어로 타이틀 + 장식 |
| 2 | Problem/Statement | `1col-center` | 히어로 타이틀 선언 |
| 3 | What Is It | `2row-30` | 소제목 밴드 + 단락+불렛 |
| 4 | How It Works | `band+col` | 소제목 밴드 + 3열 단계 |
| 5 | Key Features | `3col-equal` | 아이콘+타이틀+설명 |
| 6 | Data/Stats | `1col-full` | 큰 숫자 + 임팩트 배경 |
| 7 | Case/Evidence | `2col-50` | 사진 + 텍스트 |
| 8 | Comparison | `2x2` | 포지셔닝 맵 |
| 9 | Roadmap | `2row-50` | 상: 타이틀 / 하: 타임라인 |
| 10 | Closing/CTA | `diagonal` | 히어로 타이틀 + 액션 |

그리드 다양성: ✓ 연속 중복 없음 ✓ `2col-30`은 커버(1회)만 사용

## 실제 덱 시퀀스 패턴 분석

> 리서치 기반 (2025). 슬라이드별 그리드 구조 분석.

---

### Pattern 1 — Sequoia Capital 피치덱 템플릿 (12슬라이드)

**그리드 시퀀스**:
```
S01: 1col-full      ← 커버 (임팩트 오프닝)
S02: 1col-center    ← 문제 정의 (핵심 선언)
S03: 1col-center    ← 솔루션 (간결 선언)
S04: 2col-30        ← 시장 규모 (수치+설명)
S05: 2col-50        ← 제품 (사진+설명)
S06: 2col-30        ← 비즈니스 모델
S07: band+col       ← 트랙션 (헤딩+지표 3종)
S08: 2col-50        ← 경쟁 분석
S09: 2x2            ← 포지셔닝 맵
S10: 3col-equal     ← 팀 소개
S11: band+col       ← 로드맵 (헤딩+타임라인)
S12: 1col-full      ← 클로징 CTA
```

**패턴 요약**: `full → center(×2) → 2col/band 교차 → full`  
오프닝/클로징은 full, 메시지는 center, 증거/설명은 2col·band.  
**사용 그리드 수: 6종** — 적은 종류로 스토리 리듬을 만듦.

### Pattern 2 — Airbnb 2008 피치덱 (14슬라이드)

**그리드 시퀀스**:
```
S01: 1col-full      ← 커버
S02: 1col-center    ← 문제 (3줄 텍스트)
S03: 1col-center    ← 솔루션
S04: 1col-center    ← 시장 규모 (숫자 강조)
S05: 1col-center    ← 제품 소개
S06: 2col-50        ← 비즈니스 모델 (수익원 좌우)
S07: 1col-center    ← 시장 채택 (숫자)
S08: band+col       ← 경쟁 분석 (헤딩+2col)
S09: 2x2            ← 포지셔닝 맵
S10: 1col-center    ← 유저 성장 (차트)
S11: band+col       ← 팀
S12: 1col-center    ← 자금 사용처
S13: 1col-center    ← 요청 금액
S14: 1col-full      ← 클로징
```

**패턴 요약**: `1col-center`가 전체의 60%를 차지.  
**핵심 인사이트**: 그리드를 거의 쓰지 않고 컴포넌트(숫자/차트/텍스트)만 바꿔서 변화를 만든다.  
"단일 그리드 + 컴포넌트 변화 = 일관성 있는 스토리텔링"의 교과서.  
**사용 그리드 수: 4종**

---

### Pattern 3 — Buffer 2011 피치덱 (13슬라이드)

**그리드 시퀀스**:
```
S01: 1col-full      ← 커버
S02: 1col-center    ← 트랙션 (핵심 숫자)
S03: band+col       ← 제품 설명 (헤딩+스크린샷)
S04: 1col-center    ← 문제 정의
S05: band+col       ← 솔루션 (헤딩+설명)
S06: 2col-50        ← 비즈니스 모델 (수익 구조)
S07: 2x2            ← 경쟁 포지셔닝 (로고 배치)
S08: band+col       ← 시장 규모
S09: 1col-center    ← 성장 지표
S10: 3col-equal     ← 팀
S11: band+col       ← 로드맵
S12: 1col-center    ← 투자금 사용
S13: 1col-full      ← 클로징
```

**패턴 요약**: `1col-center`와 `band+col`이 교차하며 리듬을 형성.  
선언(center) → 설명(band) → 선언(center) → 설명(band)의 2박자 리듬.  
**사용 그리드 수: 5종**

### Pattern 4 — Apple Keynote (제품 발표)

**그리드 시퀀스 (반복 단위)**:
```
[임팩트] 1col-full     ← 제품/기능 히어로
[상세]   bento-grid    ← 6~9칸 비균등 그리드 (기능 나열)
[임팩트] 1col-full     ← 다음 기능 히어로
[상세]   bento-grid    ← 또 다른 조합
[선언]   1col-center   ← 핵심 메시지 ("One more thing")
...
```

**패턴 요약**: `full ↔ bento-grid` 2박자를 30~50슬라이드 반복.  
**핵심 인사이트**: 그리드는 2종뿐. 변화는 전부 컴포넌트(사진/동영상/숫자/텍스트).  
Bento Grid = 칸 크기가 비균등한 복합 그리드 (2×3, 1+2+1, 2+1 등).  
**사용 그리드 수: 2~3종**

### Pattern 5 — McKinsey 컨설팅 덱

**그리드 시퀀스**:
```
S01: 1col-center    ← 핵심 메시지 (Headline)
S02: band+col       ← 근거 1 (헤딩 + 차트/데이터)
S03: 2col-50        ← 비교 (좌: 현황 / 우: 제안)
S04: band+col       ← 근거 2
S05: 3col-equal     ← 3가지 권고안
S06: 2x2            ← 2×2 분석 프레임워크
S07: band+col       ← 근거 3
S08: 1col-center    ← 결론 (핵심 메시지 반복)
```

**패턴 요약**: 모든 슬라이드에 "Headline" 라인이 있음 (band+col의 상단 밴드).  
복잡한 분석도 band+col 기반으로 정리 → 한 줄 헤드라인이 결론을 담고, 본문이 근거.  
**사용 그리드 수: 5종** / 컨설팅 특성상 `2x2`와 `3col`이 분석 프레임워크로 자주 등장.

## 시퀀스 패턴 3가지 원형

리서치에서 발견된 공통 패턴:

### 원형 A — 선언·증거 2박자 (Airbnb, Buffer 스타일)
```
선언 (center/full) → 증거 (band/2col) → 선언 → 증거 → ...
```
심플한 스토리텔링. 청중이 "주장 → 근거"를 반복해서 흡수.

### 원형 B — 히어로·상세 2박자 (Apple 스타일)
```
히어로 (full) → 상세 (bento/band) → 히어로 → 상세 → ...
```
감성 우선, 이성 후. 충격 → 설명 → 충격 → 설명.

### 원형 C — 스토리 아크 (Sequoia 스타일)
```
full(오프닝) → center×N(스토리) → 2col/band×N(증거) → full(클로징)
```
오프닝과 클로징을 full로 감싸고, 중간에 메시지+증거를 쌓음.

---

## 핵심 인사이트

```
1. 성공한 덱은 그리드 종류가 적다 — 보통 3~6종
2. 단일 그리드 + 컴포넌트 변화로 다양성을 만드는 것이 고급 기법
3. 오프닝/클로징은 거의 항상 1col-full 또는 1col-center
4. 연속 슬라이드는 선언↔증거 또는 히어로↔상세로 교차하는 2박자 리듬
5. 그리드 종류를 많이 쓸수록 덱이 산만해진다
```

**에이전트 지침**: 덱 설계 시 그리드를 5종 이내로 제한하고,  
나머지 다양성은 컴포넌트(숫자/아이콘/사진/텍스트 크기)로 확보한다.

## 효과별 레이아웃 매트릭스

원하는 시각 효과 → 그리드 + 컴포넌트 추천.

| 원하는 효과 | 1순위 | 2순위 | 3순위 |
|-------------|-------|-------|-------|
| **강한 임팩트** | `1col-center` 히어로 타이틀 | `1col-center` 빅넘버 | `1col-full` 이미지 |
| **객관성** | `band+col` 차트+인사이트 | `2x2` 포지셔닝/비교 | `2col-50` 대조 |
| **신뢰감** | `band+col` KPI 대시보드 | `1col-full` 인용 | `2row-30` 타임라인 |
| **이해도** | `band+col` 다이어그램 | `3col-equal` 아이콘+라벨 | `2col-50` Before/After |
| **감정 이입** | `1col-full` 이미지 | `1col-full` 인용 풀스크린 | `2col-20` 이미지+텍스트 |
| **전문성** | `2col-20` 목업+어노테이션 | `2x2` 분석 프레임 | `band+col` 데이터 차트 |
| **분위기 전환** | `1col-full` 간지(섹션 디바이더) | `1col-center` 히어로 | `diagonal` |

## 발표 유형별 추천 시퀀스

```
01  Cover            2col-30  혼합 (다크+크림)

06  2col-20          제품 목업 (스크린샷+어노테이션)
07  band+col         성장 차트 (헤딩밴드+라인차트)
```
