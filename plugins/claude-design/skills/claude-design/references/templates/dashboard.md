# Template: Dashboard / Admin Panel

업종: SaaS 내부 화면 / 어드민 / 데이터 시각화
목적: 데이터 모니터링, 빠른 의사결정
권장 스타일 태그: `professional, minimal, technical, b2b`

---

## 섹션 구조

```
① Sidebar         — 로고 + 네비게이션 메뉴 + 사용자 프로필
② Top Bar         — 페이지 제목 + 검색 + 알림 + 기간 필터
③ KPI Cards       — 핵심 지표 4개 (수치 + 변화율 + 미니 차트)
④ Main Chart      — 주요 그래프 (2:1 비율, 라인/바/면적)
⑤ Sub Chart       — 보조 그래프 또는 분포 (파이/도넛)
⑥ Data Table      — 상세 데이터 목록 (정렬, 필터, 페이지네이션)
⑦ Activity Feed   — 최근 활동 로그 (우측 사이드바)
```

## 컴포넌트 가이드라인

| 컴포넌트 | 권장 스타일 |
|---------|------------|
| KPI 카드 | 배경 흰색 + 미묘한 border or shadow, 수치 28-32px bold |
| 차트 | Chart.js / D3 권장, 색상은 DESIGN_SYSTEM accent 기반 |
| 테이블 행 | hover시 배경 미묘하게 변화, 체크박스 선택 시 highlight |
| 상태 배지 | 활성(초록), 대기(노랑), 오류(빨강) — 색맹 대비 텍스트 병행 |
| 사이드바 | 현재 페이지 active 상태 명확히 (배경 + 좌측 border accent) |

---

## 콘텐츠 가이드라인

- **밀도**: 정보 밀도 높게, 여백 최소화 (component spacing 16px)
- **색상**: 데이터 색상 4-6개 고정 팔레트 (일관성 필수)
- **폰트**: 숫자는 tabular-nums (monospace 또는 font-variant-numeric)
- **로딩**: skeleton 상태 항상 고려 (뼈대 레이아웃)

## 브레이크포인트

```css
/* Mobile (≤767px) */  → Sidebar 숨김 (하단 탭바), KPI 2열, Table 스크롤
/* Tablet (768-1023px)*/ → Sidebar 아이콘만, KPI 4열 가능
/* Desktop (≥1024px) */ → 풀 레이아웃
```
