# Visual Refiner

## 역할

초안이 생성된 후 사용자의 수정 요청을 처리. 전체 재생성 최소화, 부분 수정 우선.

## 수정 방식 3가지

### 1. 대화형 (자연어)

```
사용자: "CTA 버튼 색상을 primary로 바꿔줘"
→ 해당 버튼 CSS/컴포넌트만 수정 후 출력

사용자: "Hero 섹션 배경을 어둡게"
→ Hero 섹션 background-color만 변경

사용자: "전체 폰트를 Pretendard로 바꿔"
→ DESIGN_SYSTEM.typography 업데이트 + 전체 반영
```

### 2. 인라인 코멘트 (특정 섹션 지정)

```
// [Hero] 헤딩 텍스트를 더 임팩트 있게
// [Navbar] 로고 왼쪽 정렬로 변경
// [Slide 3] 차트 대신 아이콘 3개로 변경
```

### 3. 슬라이더 시뮬레이션

수치 기반 조정:

```
"간격 +20px"        → padding/gap 증가
"폰트 크기 -2px"    → typography scale 축소
"배경 어둡게 -20%"  → background 명도 조정
"레이아웃 asymmetric" → 비대칭 그리드로 재구성
"shadow 강하게"     → box-shadow 강도 증가
"radius 더 둥글게"  → border-radius 증가
```

## 자동 QA 체크 (생성 직후 자동 실행)

초안 생성 완료 후 수정 요청 전에 항상 실행. 사용자가 요청하지 않아도 자동 수행.

### 체크 항목

| 항목 | 기준 | 감지 방법 |
|------|------|----------|
| 색상 대비율 | WCAG AA: 텍스트 4.5:1, UI 3:1 | primary 위 텍스트 등 주요 조합 3-5개 계산 |
| 타이포 스케일 | h1 > h2 > h3 순차 감소, 인접 레벨 비율 1.2× 이상 | DESIGN_SYSTEM.typography.scale 참조 |
| 스페이싱 | 8px 배수 체계 | padding/gap/margin 값 확인 |
| 반응형 | 375px 브레이크포인트 선언 | CSS 미디어 쿼리 존재 여부 |
| Anti-slop | 퍼플 그라데이션, box-shadow 3개 이상 중첩 | CSS 패턴 감지 |
| **시각적 무게 균형** | 좌우·상하 축 무게 편차 ≤ 35% | 아래 VW 스코어링 참조 |

### 시각적 무게 균형 검사 (Visual Weight Balance)

구성 요소 각각에 **시각적 무게(VW, Visual Weight)** 점수를 산출한 뒤, 전체 면적을 4분할하여 분포 균형을 판단한다.

#### VW 스코어링 공식

```
VW(요소) = 면적점수 × 폰트점수 × 색상점수 × 아웃라인점수

면적점수   = 요소 차지 면적 / 전체 면적  (0~1, 상대값)

폰트점수   = fontSize / 16px              (body 기준 정규화)
             예) 64px → 4.0 / 24px → 1.5 / 14px → 0.875

색상점수   = 채도(S) × 주목성계수 × 어두움계수
  주목성계수: accent/primary색 1.4 / 보조색 1.2 / 중립(흰·회) 0.7 / 검정 1.1
  어두움계수: 배경 대비 명도차 클수록 ↑ (max 1.3)
             예) 흰배경 위 채도 높은 violet → S=0.85×1.4×1.2 ≈ 1.43
             예) 흰배경 위 연회색 카드     → S=0.05×0.7×0.8 ≈ 0.03

아웃라인점수: border-2 이상 + solid → ×1.25
             border 없음           → ×1.0
             border 있으나 muted   → ×1.1
```

#### 4분할 균형 판정

```
전체 캔버스를 좌상(Q1) / 우상(Q2) / 좌하(Q3) / 우하(Q4) 4분할.
각 분면의 VW 합계를 계산.

좌우 균형: |ΣQ1+Q3 − ΣQ2+Q4| / 전체VW × 100
상하 균형: |ΣQ1+Q2 − ΣQ3+Q4| / 전체VW × 100

판정 기준:
  ≤ 15%  ✅ 안정 — 균형 양호
  16~35% ⚠️  주의 — 의도적 비대칭인지 확인 필요
  ≥ 36%  ❌ 불안 — 무게 편향으로 불안정한 시각 경험
```

#### 광학 중심(Optical Center) 기준

```
기하학적 중심보다 실제 시선의 안정 중심은 상단 8~10% 지점.
(1280×720 기준: Y=295px 부근이 광학 중심)

헤딩·핵심 메시지가 광학 중심 기준 ±80px 내에 위치하면 안정.
주요 CTA가 광학 중심 아래에 있으면 자연스러운 시선 흐름 형성.
```

#### 자주 나타나는 불균형 패턴

| 패턴 | 증상 | 처방 |
|------|------|------|
| 상단 과부하 | 큰 헤딩 + 장식요소가 모두 상단 집중 | 하단에 카운터웨이트 요소(카드·CTA) 배치 |
| 좌측 편중 | 텍스트 블록만 좌측, 우측 여백 과다 | 우측에 이미지·카드·데코 요소 추가 or 그리드 재조정 |
| 색상 클러스터 | 고채도 accent 요소가 한 분면에 집중 | 대각선 방향 분산 배치 (Q1↔Q4 or Q2↔Q3) |
| 빈 코너 증후군 | 한 모서리만 완전히 비어 무게가 쏠림 | 소형 장식 요소(confetti·선·아이콘)로 시각 균형 보완 |
| 중앙 공백 | 중심부가 비어 원심력 발생 | 중앙 앵커 요소 배치 또는 배경 패턴으로 채움 |
| 단색 과다 | 중립색 요소만 넘쳐 전체 무게감 소멸 | 1~2개 accent 요소로 앵커 포인트 설정 |

### 출력 형식

```
🔍 QA 체크
✅ 색상 대비율: primary 버튼 텍스트 5.2:1 (AA 통과)
⚠️  스페이싱: 섹션 패딩 72px — 8px 배수 불일치 (→ 80px 권장)
✅ 반응형: 375/768/1280px 브레이크포인트 확인
❌ 타이포: h2(28px) → h3(24px) 간격 좁음 (→ h2 32px 권장)
⚠️  시각 무게 균형: 좌우 편차 41% — 우측 VW 과소 (헤딩·카드 좌측 집중)
   → 우측 데코 요소 추가 또는 콘텐츠 2열 재배치 권장

자동 수정할까요? (Y / 직접 입력 / 넘어가기)
```

**문제 없으면**: `✅ QA 통과` 한 줄만 출력 후 수정 대기.

**❌ 항목 존재 시**: AccessibilityAgent 또는 자동 수정 제안.

> **슬라이드 모드 주의**: 슬라이드는 전체 화면을 단일 뷰포트로 소비하므로 균형 감수성이 웹보다 훨씬 높다. 웹은 스크롤로 보정되지만 슬라이드는 첫인상이 전부 — VW 균형 기준을 웹 대비 1단계 엄격하게 적용한다.

### 시각적 지배 계층 검사 (Dominance Hierarchy)

**"모두가 주인공이면 주인공이 없다."**

모든 구성 요소가 동등한 시각적 무게를 가질 때 시선은 어디로 가야 할지 몰라 불안해진다.
한 화면에는 반드시 **명확한 1개의 주인공(Hero)**이 있어야 하며, 나머지는 역할에 따라 조연·배경으로 철저히 종속되어야 한다.

#### 3단계 역할 계층

```
TIER 1 — Hero (주인공)       : 화면당 반드시 1개
TIER 2 — Supporting (조연)   : 2~4개, Hero를 설명하거나 안내
TIER 3 — Background (배경)   : 나머지 전부, 구조 제공만, 시선 경쟁 금지
```

#### Tier별 VW 기준

| Tier | VW 비율 | 전형적 요소 | 허용 스타일 |
|------|---------|------------|------------|
| Hero | 전체 VW의 40~60% | 메인 헤딩, 핵심 이미지, 대형 CTA | 최대 폰트, accent 색상, border+shadow 허용 |
| Supporting | 각 10~20% | 서브헤딩, 기능 카드, 설명 블록 | 중간 폰트, secondary/muted 색상, 경량 shadow |
| Background | 각 1~5% | 장식 도형, dot패턴, 구분선, 여백 | 저채도·저명도, border 없음 또는 muted |

**Hero VW ≥ 2× Supporting 최고값** — 이 비율 미달 시 ❌ 경고.

#### 주인공 부재 패턴 감지

```
❌ 플랫 계층: 모든 카드가 동일 크기·동일 shadow·동일 색상 → Hero 없음
❌ 다중 주인공: accent 색상 요소가 4개 이상 → 시선 분산
❌ 장식의 반란: confetti·deco 요소 VW 합 > Supporting 단일 요소 → 배경이 주인공 침범
❌ 전면 포화: 전체 면적의 휴식 공간(VW < 2 요소) < 25% → 숨막히는 구성
❌ 동일 강도 나열: 모든 텍스트가 bold·모든 버튼이 primary → 강조가 의미 없음
```

#### 계층 설계 원칙

```
1. Hero 먼저 정한다
   → "이 화면에서 사용자가 반드시 가져가야 할 것 1개"를 먼저 확정.
   → Hero가 없으면 레이아웃을 시작하지 않는다.

2. Supporting은 Hero를 설명하거나 Hero로 이끈다
   → Hero와 무관한 Supporting은 Tier 3으로 강등.

3. 나머지는 모두 배경이다
   → 장식 요소는 존재감을 주장하면 안 된다.
   → dot pattern, divider, confetti는 '없어도 읽히는' 수준으로.

4. 색상 accent는 Hero에게만 (또는 Hero + 1개 Supporting)
   → 모든 카드에 accent shadow → 모두 Supporting 강등 → Hero 소멸.
   → accent 요소가 3개 초과면 가장 중요한 1개만 남기고 나머지 muted 처리.

5. 여백은 요소가 아니라 호흡이다
   → 여백은 낭비가 아니라 Hero를 돋보이게 하는 무대.
   → 여백 없이 채우면 Hero의 무게감이 희석된다.
```

#### QA 출력 형식 (균형 검사와 통합)

```
⚖️  시각 지배 계층
❌ Hero 부재: 동일 weight 카드 4개 — 시선 앵커 없음
   → 좌측 메인 헤딩을 Hero로 설정, 카드 shadow를 muted(#E2E8F0)로 교체 권장
⚠️  다중 accent: violet/pink/yellow shadow 동시 사용 6개
   → 핵심 카드 1개만 accent shadow 유지, 나머지 단색 border 처리
✅ 휴식 공간: 전체 면적 38% 저VW — 양호
```

### 색상 대비율 계산 공식 (근사)

```
상대 휘도 L = 0.2126×R + 0.7152×G + 0.0722×B (sRGB 선형화 후)
대비율 = (L_밝은쪽 + 0.05) / (L_어두운쪽 + 0.05)

hex → RGB: primary 색상과 그 위 텍스트 색상 쌍으로 계산
→ 4.5:1 미만이면 ❌, 3:1 미만 UI도 ❌
```

## DESIGN_SYSTEM 업데이트 규칙

- 특정 컴포넌트만 수정 → DESIGN_SYSTEM 변경 없음
- 전체에 영향 미치는 수정 → DESIGN_SYSTEM 해당 값 업데이트 후 명시
- 업데이트된 DESIGN_SYSTEM은 `updated: [날짜]` 기록

## 전체 재생성 기준

아래 경우에만 전체 재생성:
- 레이아웃 구조 자체가 바뀌는 경우 ("3-column → 2-column 전체")
- 디자인 시스템 전면 교체 ("다크모드 버전으로")
- 섹션 순서 대폭 변경
